#### IsoView

- Segmentation Workflow using Cellpose
    - Mask with threshold widget, crop, run model
    - 2D + stitching via intersection-over-union: ~77s/timepoint, unreliable segmentation
    - 3D Cellpose GPU-memory intensive and very slow, ~45min/timepoint minimum
      - 3D gives masks significantly more visually accurate, less artifacts, follows cell shape in Z more closely
    - Cells currently not tracked across time, although cellpose reports this as a capability
    - Masks are very cylindrical, could use smoothing or interpolation
    - Cellpose struggles on densely packed cells
- Do we want to change the current fg/bg mask process
  - Each pipeline function, `correct_stack` + `multi_fuse`, checks `segment_mode` (0=no masks, 1=generate+save)
  - Each pipeline function checks `apply_segmentation_mask`, if True = apply
  - Do we want instead `segment_mode=3` to generate/apply
  - `apply_segmentation_mask` is a flag that can be used for other functions, e.g. after multi-fusion you may want additional cropping/more aggressive threshold. Maybe you want beads for multi-fuse but `not` for cellpose... easier flag to understand
- Now version controlled, `pip install isoview`, v0.1.2
