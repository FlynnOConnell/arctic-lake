    - wk expects its own datasource-properties.json, which isoview doesn't generate
    - currently `uv run isoview ome2wk path/to/tiff  path/to/save` produces a `webknossos`-compatible zarr v3
  - https://webknossos.org/datasets/SPM00_TL1_CM00_VW00_corrected_pyramids_zarr-converted-69a0fb23010000e40a7ab936/view#751,1233,199,0,3.709
