- Toward fully reversable `correct_stack`
  - Dead-pixel correction is not currently reversable
      - pixels that are outside std/mean range overwritten with median-filtered values, originals not stored
      - fix: save original values at flagged (x,y,z) locations before overwriting, e.g. as a sparse array
  - Foreground segmentation zeros out background voxels via binary mask multiplication
      - `multi_fuse` already generates masks if none exist, could change `segment_mode` in `correct_stack` to instead not apply masks, just optionally generate
- Background subtraction in Isoview pipeline
    - `backgroundValues` in `clusterPT` is not image background subtraction
    - scalar per-camera dark offset, used only inside dead-pixel detection to normalize the mean projection before computing relative distances
    - actual background subtraction happens in `localEC->estimateCorrection.m` using corners of FOV
