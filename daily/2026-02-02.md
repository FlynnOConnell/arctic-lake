# Multi-Fuse Pipeline

Fuses opposing camera views into single volumes, and opposing channels as well.

Fused volumes are aligned (registration) and smoothed with blending.

## Pipeline Overview

| Step | Function | Purpose |
|------|----------|---------|
| 1 | `multi_fuse()` | Entry point, iterates timepoints and fusion pairs |
| 2 | `_find_volume_file()` | Locate input KLB/TIF files |
| 3 | `create_fusion_mask()` | Generate binary masks from volumes |
| 4 | `estimate_registration()` | Find XY alignment between views |
| 5 | `estimate_intensity_correction()` | Compute brightness matching factor |
| 6 | `apply_registration()` | Transform second volume to align with first |
| 7 | `create_slice_mask()` | Build blending transition topology |
| 8 | `blend_views()` | Merge aligned volumes |
| 9 | `write_volume()` | Save fused output |

### 1. Entry Point
```python
multi_fuse(config, estimate_params=True, apply_fusion=True)
```
Iterates over timepoints and camera pairs, calling `_fuse_camera_pair_klb()` for each.

### 2. Volume Loading
Loads 3D volumes from `output_dir/TM{tp}/SPM{sp}_TM{tp}_CM{cam}_CHN{ch}.klb`

Data shape: `(Z, Y, X)` uint16

### 3. Mask Creation
```python
mask = create_fusion_mask(vol, config.mask_percentile)
```
- Compute background: `min_val = percentile(vol, mask_percentile)`
- Compute foreground mean: `mean_val = mean(vol[vol > min_val])`
- Threshold: `mask = vol > (min_val + mask_factor * (mean_val - min_val))`
- Apply morphological closing, remove small objects

### 4. Registration Estimation
```python
transform = estimate_registration(vol0, vol1, mask0, mask1, method='phase_cross_correlation')
```
- Phase correlation in frequency domain with subpixel accuracy (upsample=10)
- Returns `{'x_offset', 'y_offset', 'correlation', ...}`

### 5. Intensity Correction
```python
correction = estimate_intensity_correction(vol0, vol1, mask0, mask1)
```
- Extract overlap region pixels
- Subtract background percentile
- Compute intensity ratio between views
- Returns `{'factor', 'operation', ...}`

### 6. Apply Transforms
```python
vol1_transformed = apply_registration(vol1, transform)
vol1_corrected = apply_intensity_correction(vol1_transformed, correction)
```
Shifts volume by offsets using `scipy.ndimage.shift`, applies intensity scaling.

### 7. Blending Mask Creation

```python
slice_mask = create_slice_mask(vol, mask)  # (Y, X) with Z transition values
avg_mask = create_average_mask(slice_mask0, slice_mask1, mode="overlap")
```
For each (y, x), computes weighted average Z position where mask is active.

### 8. Blending

```python
fused = blend_views([vol0, vol1_corrected], [avg_mask, avg_mask],
                    method=config.fusion_type, blending_range=config.fusion_blending_range[1])
```

Adaptive blending at transition plane z0:
- Front zone `(z0 - range, z0)`: linear blend vol0 → 50/50
- Back zone `(z0, z0 + range)`: linear blend 50/50 → vol1
- Outside: single best view

### 9. Output
Saves to `output_dir/fused/TM{tp}/SPM{sp}_TM{tp}_CM{cam0}_CM{cam1}_CHN{ch}.klb`

## Parameters

| Parameter | Default | Effect |
|-----------|---------|--------|
| `timepoints` | None (all) | Which timepoints to process |
| `fusion_camera_pairs` | [(0,1), (2,3)] | Camera combinations to fuse |
| `mask_percentile` | 1.0 | Background threshold for mask creation |
| `fusion_search_offsets_x/y` | (-50, 50, 10) | Registration search grid (discrete method only) |
| `fusion_flip_h/v` | False | Flip second view before registration |
| `background_percentile` | 5.0 | Background for intensity correction |
| `subsample_factor` | 100 | Speed vs accuracy for intensity calc |
| `fusion_blending_range` | (20, 20) | Transition zone width in Z-planes (ch, cam) |
| `fusion_type` | "adaptive_blending" | Blending algorithm |
| `output_format` | "klb" | Output file format |


## Notes

The inputs and outputs are confusing. Each step should have "input_dir" and "output_dir", not "output_dir" as the input dir.
