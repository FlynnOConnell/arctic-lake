# BDV affine matrices for orthogonal view registration

## format

the BDV affine is a 3x4 matrix mapping voxel indices to physical coordinates.
each row is one output axis (X, Y, Z), the 4th column is translation.
written as 12 space-separated values in row-major order:

```
row0: [a00 a01 a02 tx]   -> X output
row1: [a10 a11 a12 ty]   -> Y output
row2: [a20 a21 a22 tz]   -> Z output
```

BDV voxel indices are (X, Y, Z) not ZYX like numpy.
`size="1848 768 568"` means X=1848, Y=768, Z=568.

## dataset parameters

- pixel_xy = 0.4 um (6.5 um camera / 16x objective)
- z_step = 0.406 um
- volume shape: 568 x 768 x 1848 (Z, Y, X in numpy; X=1848, Y=768, Z=568 in BDV)
- setup 0: CM00+CM01 (z-scan), identity calibration
- setup 1: CM02+CM03 (y-scan), needs orientation transform

translation offsets used below:
- X extent: (1848 - 1) * 0.325 = 599.4 um
- Z extent: (568 - 1) * 0.406 = 230.2 um

## identity calibration (setup 0, no rotation)

```
0.325 0.0 0.0 0.0 0.0 0.325 0.0 0.0 0.0 0.0 0.406 0.0
```

voxel_X * 0.325 -> physical_X, etc. both setups currently use this.

## candidate transforms for setup 1

### 1. 90-degree rotation around Y axis (X -> Z, Z -> -X)

```
0.0 0.0 0.406 0.0 0.0 0.325 0.0 0.0 -0.325 0.0 0.0 599.4
```

rotates volume 90 deg in XZ plane. translation 599.4 keeps volume in positive coords.

### 2. 90-degree rotation around Y axis (opposite: X -> -Z, Z -> X)

```
0.0 0.0 -0.406 230.2 0.0 0.325 0.0 0.0 0.325 0.0 0.0 0.0
```

same rotation, opposite direction. translation 230.2 compensates the flip.

### 3. flip Z only (reverse Z scan direction)

```
0.325 0.0 0.0 0.0 0.0 0.325 0.0 0.0 0.0 0.0 -0.406 230.2
```

mirrors along Z axis. use if volumes are Z-flipped relative to each other.

### 4. flip X only

```
-0.325 0.0 0.0 599.4 0.0 0.325 0.0 0.0 0.0 0.0 0.406 0.0
```

mirrors along X axis.

### 5. swap X and Z (matches orient_vw90: transpose + flip)

```
0.0 0.0 0.406 0.0 0.0 0.325 0.0 0.0 0.325 0.0 0.0 0.0
```

X reads from voxel Z, Z reads from voxel X, both positive.
this is the orient_vw90() axis swap from the isoview codebase. try this first.

### 6. swap X and Z with Z flip

```
0.0 0.0 0.406 0.0 0.0 0.325 0.0 0.0 -0.325 0.0 0.0 599.4
```

same as #5 but flips the Z output. try if #5 looks mirrored.

## how to apply in bigstitcher

1. right-click selected views > Registration > Apply Transformations
2. select setup 1 only, all timepoints
3. choose "Affine" model
4. paste the single line of 12 values
