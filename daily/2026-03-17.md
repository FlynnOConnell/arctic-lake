## Notes

Matrix conventions across the three codebases:

Column 1: Storage
BigStitcher (BDV XML): Row-major 3×4, 12 values in <affine> tag
diSPIMFusion (.tmx): Row-major 4×4, tab-delimited .tmx file
regDeconProject (MATLAB): Row-major 4×4 flat array via libpointer('singlePtr', Tmx)
────────────────────────────────────────
Column 1: Coordinate order
BigStitcher (BDV XML): XYZ (world space)
diSPIMFusion (.tmx): XYZ (isotropic voxels)
regDeconProject (MATLAB): XYZ (MATLAB's ReadTifStack convention)
────────────────────────────────────────
Column 1: Convention
BigStitcher (BDV XML): Forward: p_world = T @ p_raw
diSPIMFusion (.tmx): Inverse: for output voxel p, sample source at TMX @ p
regDeconProject (MATLAB): Forward: reg3d outputs a matrix that is both input seed and result;
  affine3d(T) uses MATLAB's row-vector convention (p' = p * T) so T is the transpose of the
  column-vector form
────────────────────────────────────────
Column 1: Transform chain
BigStitcher (BDV XML): Outermost listed first: compose left-to-right T1 @ T2 @ ... @ Tn
diSPIMFusion (.tmx): Single pre-composed matrix
regDeconProject (MATLAB): Single matrix (identity default, updated by reg3d)

Conjugation pipeline (BDV → diSPIMFusion):
1. Parse XML, compose per-setup: T_full = T1 @ T2 @ ... @ Tn
2. Relative transform: T_rel = inv(T_full_spima) @ T_full_spimb (SPIMB→SPIMA in raw XYZ voxels)
3. Isotropize via conjugation: T_rel_iso = S @ T_rel @ inv(S) where S = diag(1, 1, pz/pxy, 1)
4. Build centered rotation: R_centered = T(c) @ R90Y @ T(-c) where c = (n-1)/2 of isotropized SPIMB
5. Compose in inverse convention: TMX = R_centered @ inv(T_rel_iso)

Conjugation pipeline (BDV → scipy):
1. Same steps 1-2
2. Axis permutation (BDV XYZ → numpy ZYX): T_zyx = P @ T_rel @ P where P swaps axes 0↔2
3. Pass inv(T_zyx) to scipy.ndimage.affine_transform (which uses inverse/pull convention)

Conjugation pipeline (BDV → regDeconProject/MATLAB):
1. Same steps 1-2 to get T_rel (3×4 or 4×4, column-vector convention)
2. MATLAB affine3d expects row-vector form: T_matlab = [R' zeros(3,1); t' 1] — i.e., transpose the
3×3 block, put translation in row 4 (see orient_90deg_angle.m lines 8-11)
3. imwarp(stackB, inRefB, tform, ...) applies the forward transform with optional imref3d for
physical coordinates

Key gotcha: the step-3 conjugation S @ T @ inv(S) is required whenever converting between
anisotropic and isotropic spaces. Forgetting it causes the translation components to be wrong by
the Z-ratio factor. Similarly, the P @ T @ P conjugation for numpy is easy to forget since it only
affects off-diagonal rotation terms and Z-translations.
