# Processing steps in `view_suite2p_volume.py`

## 1. ΔF/F computation — `_load_dff`

**Input**: `F.npy` (raw fluorescence, shape `(N_rois, T)`), `Fneu.npy` (neuropil, same shape).

**Output**: `dff` array, same shape, used as the trace per ROI.

**Formula** (suite2p convention):

$$F_{\text{corr}} = F - 0.7\,F_{\text{neu}}$$

$$\text{baseline} = \mathrm{percentile}_{20}(F_{\text{corr}}, \text{axis}=\text{time})$$

$$\Delta F/F = \frac{F_{\text{corr}} - \text{baseline}}{|\text{baseline}| + 10^{-6}}$$

Skipped if a precomputed `dff.npy` is on disk — that file is loaded directly.

---

## 2. Per-plane projection assembly — `load_volume`

**Input**: each plane's `ops.npy` providing `max_proj` (cropped to `yrange`/`xrange`) or fallback `meanImgE`/`meanImg`; plus `stat.npy`/`iscell.npy`.

**Output**: `proj_vol` of shape `(Z, Y, X)`; list of `Roi` records carrying `(z, ypix, xpix, prob, skew, trace, ...)`.

**Formula**: per plane,

$$\text{proj\_vol}[z,\, y_0{:}y_0{+}H,\, x_0{:}x_0{+}W] = \text{max\_proj}_{z}$$

where $(y_0, x_0) = (\text{yrange}[0], \text{xrange}[0])$. ROI pixels outside the FOV are filtered: $(y\text{pix} \in [0, L_y)) \land (x\text{pix} \in [0, L_x))$.

---

## 3. Cross-plane ROI merging (IoU + union-find) — `merge_adjacent_planes`

**Input**: list of `Roi`, `iou_thresh`, `max_dz`, `centroid_radius` prefilter.

**Output**: list of `MergedRoi` clusters; each holds its members, $\max(\text{prob})$, and the set of z-planes covered.

**Formula**: for every candidate pair $(i, j)$ on planes $z_i, z_j$ with $0 < z_j - z_i \le \text{max\_dz}$ and centroid distance $\lVert \text{med}_i - \text{med}_j \rVert \le \text{centroid\_radius}$ (XY only):

$$\text{IoU}(i, j) = \frac{|P_i \cap P_j|}{|P_i \cup P_j|},\quad P_k = \{(y, x) : k\text{'s pixels}\}$$

Pairs with $\text{IoU} \ge \text{iou\_thresh}$ are merged via union-find (path compression). Connected components → clusters.

---

## 4. 3D ellipsoid fit per cell — `_fit_ellipsoid`

**Input**: list of `Roi` (one cell's members), volume `shape`, anisotropic `scale = (s_z, s_y, s_x)`, `sigma_thresh` (radius in std devs).

**Output**: tuple `(bbox_voxel_slices, inside_bool_mask)` — voxel bbox containing the iso-surface plus the bool mask of voxels inside it.

**Formulas**:

Convert pixel coords to physical: $p_k = (z_k s_z,\ y_k s_y,\ x_k s_x)$. Then

$$\mu = \tfrac{1}{N} \sum_k p_k,\qquad \Sigma_0 = \tfrac{1}{N-1} \sum_k (p_k - \mu)(p_k - \mu)^{\top}$$

Diagonal regularizer so single-plane ROIs aren't singular in Z:

$$\Sigma = \Sigma_0 + \mathrm{diag}\big((0.5\,s_z)^2,\ (0.5\,s_y)^2,\ (0.5\,s_x)^2\big)$$

A voxel at physical position $p$ is **inside** the ellipsoid iff the squared Mahalanobis distance is at most $\sigma_{\text{thresh}}^2$:

$$d^2(p) = (p - \mu)^{\top} \Sigma^{-1} (p - \mu) \;\le\; \sigma_{\text{thresh}}^2$$

Bounding box: $\mu_{\text{vox}} \pm r_{\text{vox}}$ where

$$r_{\text{vox},i} = \frac{\sqrt{\lambda_{\max}(\Sigma)} \cdot \sigma_{\text{thresh}}}{s_i}$$

— a conservative axis-aligned cap using the largest principal radius. Inside that bbox, $d^2$ is evaluated on a physical-units meshgrid and thresholded.

---

## 5. Painting into the 3D label volume — `render_masks_flat` / `render_masks_merged` / `render_ellipsoids`

**Input**: ROIs / clusters / ellipsoids + the iscell-probability `threshold`.

**Output**: `(Z, Y, X)` `uint32` array with **stable label IDs** $=$ `index + 1`. Stability matters so picked label → ROI index lookup survives threshold drag.

**Formula**: for each entry above threshold,

$$\text{mask}[z, y, x] = \text{index} + 1 \quad \text{for } (y, x) \in \text{member pixels (or ellipsoid \texttt{inside})}$$

In the ellipsoid path, voxels with `mask == 0` are written first (first-written-wins for overlapping ellipsoids).

---

## 6. Lazy 4D movie stack — `load_movie_4d`

**Input**: each plane's `data.bin` (registered `int16`, shape $(T_p, L_{y,p}, L_{x,p})$); plus `ops['nframes']`, `Ly`, `Lx`.

**Output**: dask array of shape $(T, Z, Y, X)$ where

$$T = \min_p T_p,\qquad Y = \max_p L_{y,p},\qquad X = \max_p L_{x,p}$$

chunked `(1, 1, Y, X)` so napari only reads the visible z-plane × T slice.

**Operation**: memory-map each `data.bin` (zero copy), pad mismatched planes with zeros to `(Y, X)`, then `da.stack(planes, axis=1)`. No data is read until napari requests a slice.

---

## 7. Trace cursor ↔ T slider sync

**Input**: napari's `viewer.dims.events.current_step` (fires when any slider moves).

**Output**: matplotlib `axvline` x-position on the trace plot.

**Formula**: cursor at frame

$$t = \mathrm{clip}\big(\text{current\_step}[0],\; 0,\; T_{\text{trace}} - 1\big)$$

---

## Pipeline order

At startup, in this order:

1. Load planes → projections + ROIs + traces (steps 1, 2)
2. Optional IoU merge (step 3)
3. Optional ellipsoid fit (step 4)
4. Render label volume (step 5)
5. Load lazy movie (step 6)
6. Wire napari layers, dock controls, click handler, T-cursor sync (step 7)

The threshold slider only re-runs step 5; everything else is precomputed.
