### Segmentation Mask Workflow

1. Run clusterPT with segmentMode=2
2. Run clusterPT again to generate global masks
3. Run clusterPT a third time to apply to remaining image stacks

-> Python accomplishes this in a single run!

When would we want to use Global segmentation mode?

segment_mode=0: No segmentation/masking
segment_mode=1: Segment + apply mask to output (removes background)
segment_mode=2: Generate masks only (saves .segmentationMask.klb files, doesn't apply to output)
segment_mode=3: Apply a pre-existing global mask (reads from global_mask_dir)

Params that effect the segmentation masking:

- Gaussian kernel size	gauss_kernel	Larger = more smoothing, fewer small features
- Gaussian sigma	gauss_sigma	Larger = more blur, smoother boundaries
- Threshold	segment_threshold	Lower = more permissive (more foreground), Higher = stricter
- Mask percentile	mask_percentile	Percentile for minimum intensity calculation

To remove small "dots" in the background:
- Increase gauss_sigma (e.g., 2.0 → 3.0) - more smoothing before thresholding
- Increase segment_threshold (e.g., 0.4 → 0.5) - stricter threshold removes weak features
- Add morphological cleanup - the code in masks.py has apply_bwareaopen() but it's only used in the fusion pipeline

references and dependents parameters
Purpose: Channel grouping for segmentation mask sharing

How they work:

references: matrix where each row defines a group of channels that generate the segmentation mask
dependents: matrix where each row defines channels that use the mask from the corresponding references row but don't contribute to generating it
Default behavior (lines 282-288):


```python
if isempty(references)
    references = channels';  % each channel is its own reference
    dependents = [];
    % thresholds expanded to match
end
```
Processing flow:

Reference channels (lines 358-536):

Loaded, transformed, corrected
Gaussian filtered → segmentation mask generated
If multiple references in same row: masks are OR'd together (lines 541-546)
Masked output saved
Dependent channels (lines 800-979):

Loaded, transformed, corrected
No segmentation mask generated - they reuse the reference mask
Masked with referenceMasks{1} (line 918) or globalMask (line 920)
Masked output saved

Example:

```python
references = [0; 1];    % two groups, CHN00 and CHN01 are references
dependents = [];        % no dependents

% OR for multi-channel imaging:
references = [0];       % CHN00 generates the mask
dependents = [1];       % CHN01 uses CHN00's mask (e.g., structural vs functional channel)
```

Why use it:

  When you have a bright structural channel and a dim functional channel, use the structural channel as reference to generate a good mask, and apply it to the functional channel as a dependent
  Saves computation (no need to run segmentation twice)
  Ensures identical masks between channels

Summary for your single-channel case:

With references = [] and dependents = [], MATLAB defaults to treating each channel as its own reference.
- For single-channel data, this has no effect. These parameters only matter for multi-channel acquisitions where you want one channel's mask applied to another.
